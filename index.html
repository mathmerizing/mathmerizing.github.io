Solving the Schrödinger Equation with deal.ii
authors:    Leon Samolik, Max Schröder, Julian Roth

This group project is the result of a lecture on programming finite elements in C++ held at the Leibniz Universität Hannover in the winter semester 2019/20. We decided to simulate the quantum harmonic oscillator numerically. For that purpose, we discretized the Schrödinger equation with the harmonic oscillator as the potential, i.e.


To apply the finite element method, we discretized the time derivative with the one step θ-scheme. The resulting equation was split into its real and imaginary part and then transformed into a weak form by multiplication with a test function and integration by parts. In a 2D setting a direct solver, like the LU decomposition, turned out to be sufficient to solve our linear equation system. Additionally, the inverse only needed to be calculated once, since the system matrix remained constant throughout time.

In the animation below we started with a normal distribution as our initial condition and applied homogeneous Dirichlet conditions. This simulates an imprisoned quantum particle. The particle is being reflected off the boundary and with time moves towards the origin where it interferes with itself.

Detecting basketballs in a video and plotting its trajectory
author:    Julian Roth

The goal of this project was to be able to analyze the release angle when shooting a basketball. Using Tensorflow's Object Detection API, I applied transfer learning to be able to detect basketballs in images. Thus I was able to go through each frame of a video and draw a bounding box around the basketball. This can be seen in the video below.


Using the center of the boxes surrounding the basketball, there are now multiple points which can be used for a quadratic regression, i.e. we can fit a parabola to the basketball's trajectory. When running the Python script, there are two sliders which can be adjusted manually to figure out the exact release angle:

  • release: Move the release point along the parabola of the basketball shot.
  • frame number: Choose the frame of the video which is being shown.

Detecting squats from the camera feed
authors:    Yerso Checya Sinti, Julian Roth

This is a working online prototype of a squat detection application. Using Posenet and a simple heuristic, we are able to count the user's squats, if he/she stays in one place.


How to use the online demo:

  1. Push the button "Try it here", which is below these instructions.
  2. Enable your browser to use the camera.
  3. Then a countdown starts and you should go to the place, in which you want to do the squats.
  4. When the countdown is done, your pose will be detected and you can do some squats.

Creating an AI for Four In a Row
authors:    Yerso Checya Sinti, Julian Roth

Starting this project, we wanted to build our frist Android app. Our initial development has been for the command line though and some features are only available in the command line version.

We implemented the popular board game "Four In A Row" and created a bot, which used Minimax and Alpha-beta pruning. The Desktop version also uses multi-threading to speed up the bots computation. A detailed explanation of our Minimax implementation and our board evaluation function can be found here.

The Minimax algorithm is sufficient to beat most human players, if the computer is calculating at least 8 plies into the future. However, it can still be defeated. Hence, we tried to create a stronger bot by implementing NEAT, a genetic algorithm. Each genome is a fully connected neural network with 42 input neurons (= number of game board entries) and 7 output neurons (= columns in which tile can be placed). The genomes play against a short-sighted Minimax bot. Throughout the training process the network grows and might eventually look like:


Although we have tried different starting positions, the NEAT algorithm wasn't able to generalize or learn useful strategies. There are simply too many weights in the neural network, which hinders the convergence of the genetic algorithm to a good bot. Therefore this type of opponent hasn't been implemented in the Android app.

For a more detailed documentation of our project please check out the project's GitHub page.
